## 决策记录（含理由）

### 决策 1：从 v1 8 步执行改成 v2 7 阶段
- **理由**：用户要求"稳定、安全、有效推进"。v1 步骤太粗、缺验证点
- **影响**：每个阶段结束都强制 `swift build` + dev-log + 用户确认。开发节奏变慢但更稳

### 决策 2：引入 `docs/` 长效规范 + `dev-log/` 每日日志
- **理由**：用户要求项目有"具体标准文件"。也便于 Claude 跨会话保持一致性
- **影响**：每次会话开始必读最新 dev-log，结束必写新条目（在 CLAUDE.md 红线中固化）

### 决策 3：技术栈坚持 Swift + SwiftUI + SwiftPM
- **理由**：用户已装完整 Xcode；MenuBarExtra 是 macOS 13+ 一行代码；性能/体积最优
- **影响**：零外部依赖，靠系统库实现全部功能

### 决策 4：用 CLI 测试模式而非单元测试
- **理由**：UI 测试在 SwiftPM 里设置复杂；用 `CANVAS_CLI=1` 环境变量分流到命令行入口，能用真实 Token 验证 API 和存储层
- **影响**：阶段 1-3 都能在不打开 GUI 的情况下端到端验证

### 决策 5：Token 必须存 Keychain
- **理由**：Token 等同密码，UserDefaults / plist 是明文。Keychain 由系统加密、按 App 隔离
- **影响**：阶段 2 必须实现 KeychainStore，不能图省事用 UserDefaults
