# 08 · E2E 验证清单

最终发布前手动跑一遍。每项打勾。

## 构建
- [ ] `swift build` 通过，无 warning
- [ ] `./build.sh` 生成 `dist/CanvasDeadlines.app`

## 首次启动
- [ ] 双击 `.app` 后，菜单栏出现图标
- [ ] Dock 中**不**出现图标（LSUIElement 生效）
- [ ] 首次点击图标显示 Onboarding 页面

## Token 配置
- [ ] Onboarding 中"打开 Canvas 网站"按钮能用浏览器打开 `https://my.uclaextension.edu`
- [ ] 粘贴 Token 后点「测试并保存」显示"✅ 已连接，发现 X 门课程"
- [ ] 关闭再打开 App → 不再显示 Onboarding，直接显示列表

## 数据加载
- [ ] 列表中能看到真实的 assignments / quizzes
- [ ] 加载中显示 ProgressView + 文案
- [ ] 加载完成 < 3 秒（课程数 ≤ 20）

## 排序与显示
- [ ] 列表确实按 `due_at` 升序
- [ ] 显示数量 ≤ 10
- [ ] 没有 due_at 的项目不出现
- [ ] 倒计时文案正确：
  - [ ] 还剩 N 天
  - [ ] 明天截止
  - [ ] 今天截止
  - [ ] 已逾期 N 天

## 过期项
- [ ] 过期项整行背景淡红
- [ ] 文字红色
- [ ] 倒计时显示「已逾期 N 天」

## 视图切换
- [ ] 点 `[按学科]` → 列表按课程分组
- [ ] Section header 显示课程代码
- [ ] 点 `[全部]` → 回到扁平列表

## 跳过流程
- [ ] 点某项右侧「跳过」→ 该项立刻从列表消失
- [ ] 打开设置 → 已跳过列表中能看到该项
- [ ] 点设置中的「恢复」→ 该项回到主列表

## 标记提交流程
- [ ] 点某项右侧「标记提交」→ 该项立刻消失
- [ ] 在 Canvas 网页里查看该项 → 仍然是"未提交"（本地标记不上传）

## 自动检测提交
- [ ] 在 Canvas 网页里真正提交一项
- [ ] 回到 App 点 🔄 刷新
- [ ] 该项自动消失

## 持久化
- [ ] 退出 App → 重新打开 → Token 仍在（不需要重新输入）
- [ ] 退出 App → 重新打开 → 跳过列表仍在

## 错误处理
- [ ] 把 Token 故意改错 → 点刷新 → 显示"Token 失效或已过期"
- [ ] 断网 → 点刷新 → 显示"请检查网络"
- [ ] 错误状态下「打开设置」按钮可用

## 退出
- [ ] 菜单底部点「退出」→ 进程结束 → 菜单栏图标消失
